Modulos Agregados Por Admin

Dinamico

Paso a Paso: Acceso a Almera

Documentado
  1. Abrir el enlace de Almera SGI desde el modulo.
  2. Confirmar que la conexion sea sgifunluker.
  3. Ingresar correo de soporte y clave operativa.
  4. Validar que cargue el panel de seguimiento sin errores.

Paso a Paso: Abrir Formulario y App Script

Clave
  1. Abrir el enlace de Google Form en modo edicion.
  2. Iniciar sesion con la cuenta autorizada para el formulario.
  3. En el formulario, ir a Extensiones y seleccionar Apps Script.
  4. Verificar scripts, activadores y ultima ejecucion.
  5. Si modificas algo, guardar, probar envio y revisar salida en ClickUp.

Paso a Paso: Gestion de Solicitudes en ClickUp

Operacion
  1. Abrir el tablero de ClickUp del area de soporte.
  2. Filtrar por solicitudes nuevas y prioridad.
  3. Asignar responsable y fecha compromiso.
  4. Actualizar estados: Nuevo, En proceso, Resuelto.
  5. Registrar comentario final y evidencias de solucion.

Paso a Paso: Configurar API de ClickUp

Integracion
  1. Entrar a ClickUp con la cuenta administradora del espacio.
  2. Ir a Settings > Apps (o Integrations) y generar el API Token.
  3. Copiar el token una sola vez y guardarlo en gestor de secretos interno.
  4. En Vercel, abrir el proyecto y crear variable CLICKUP_API_TOKEN para Production, Preview y Development.
  5. Guardar tambien CLICKUP_LIST_ID y CLICKUP_TEAM_ID como variables de entorno.
  6. Probar con solicitud de prueba y validar que cree/actualice tarea en la lista correcta.

Donde debe estar: el token debe vivir en variables de entorno del backend, nunca en HTML ni JavaScript del cliente.

Runbook de Resolucion API ClickUp

SRE Soporte
1. Detectar

Identifica codigo HTTP, endpoint y hora exacta del fallo.

2. Aislar

Valida token, IDs y payload antes de tocar produccion.

3. Corregir

Aplica el flujo del error especifico y vuelve a probar.

4. Escalar

Si no resuelve en 20 min, escalar con evidencia y logs.

  1. Confirmar que estas usando el enlace correcto de SGI.
  2. Verificar la conexion sgifunluker en la pantalla de acceso.
  3. Validar correo de soporte y clave actualizada.
  4. Si persiste, escalar al administrador de plataforma.
  1. Verifica que el backend lea CLICKUP_API_TOKEN desde Vercel y no desde frontend.
  2. Prueba token con endpoint de usuario de ClickUp para validar autenticacion.
  3. Si falla, genera token nuevo en ClickUp (Settings > Apps/Integrations).
  4. Actualiza variable en Vercel para Production, Preview y Development.
  5. Haz redeploy y reintenta envio desde formulario controlado.

Escalar: Si persiste tras regenerar token, abrir ticket con ClickUp Support con request id y timestamp.

  1. Confirma que la cuenta dueña del token pertenece al Workspace correcto.
  2. Verifica rol sobre el Space y List objetivo (Editor/Admin recomendado).
  3. Confirma que no exista restriccion por equipo o carpeta privada.
  4. Prueba crear tarea manual con esa cuenta dentro de la lista.
  5. Actualiza permisos y reintenta integracion desde Apps Script/backend.

Escalar: Si la cuenta ve la lista pero API sigue bloqueando, escalar a administrador de Workspace.

  1. Revisa si se usa el endpoint correcto segun version API de ClickUp.
  2. Valida CLICKUP_LIST_ID y CLICKUP_TEAM_ID en Vercel.
  3. Confirma que los IDs no tengan espacios ni caracteres ocultos.
  4. Haz consulta GET al recurso antes de crear o actualizar tareas.
  5. Corrige ruta y vuelve a ejecutar prueba de punta a punta.

Tip: Mantener un archivo interno con IDs oficiales y fecha de ultima validacion.

  1. Confirma en logs cuantas peticiones por minuto se estan enviando.
  2. Implementa reintentos con backoff exponencial: 2s, 4s, 8s, 16s.
  3. Evita enviar eventos duplicados desde Forms/App Script.
  4. Agrupa operaciones por lote cuando aplique.
  5. Monitorea 24h para confirmar estabilidad sin nuevos 429.

Escalar: Si el volumen es legitimo de negocio, evaluar upgrade o arquitectura con cola.

  1. Repite request de prueba para descartar glitch puntual.
  2. Verifica estado de servicios de ClickUp y latencia de red.
  3. Activa circuito de reintento con maximo 3 intentos controlados.
  4. Si hay datos criticos, usa cola temporal y reprocesa luego.
  5. Documenta incidente con hora inicio/fin e impacto.

Escalar: Si dura mas de 15 min, escalar a lider y soporte de proveedor.

  1. Verifica URL publica activa y accesible desde internet (sin bloqueo IP).
  2. Confirma metodo HTTP esperado y ruta correcta en backend.
  3. Valida firma/secret usando CLICKUP_WEBHOOK_SECRET.
  4. Revisa logs del receptor: status, body y headers recibidos.
  5. Regenera webhook en ClickUp si detectas configuracion corrupta.

Escalar: Si ClickUp marca entregado pero backend no recibe, revisar firewall y CDN.

  1. Captura payload completo saliendo de Apps Script antes del POST.
  2. Valida mapeo de campos: titulo, descripcion, prioridad, responsable.
  3. Aplica valores por defecto para campos obligatorios faltantes.
  4. Registra validacion previa y rechaza envio incompleto con mensaje claro.
  5. Prueba con 3 casos: completo, parcial y datos invalidos.

Escalar: Si el mapeo falla intermitente, versionar script y comparar ejecuciones.

  1. Intentar recuperacion por correo alterno autorizado.
  2. Si no hay acceso, solicitar reset al responsable del sistema.
  3. Actualizar registro interno de cambio de clave.
  4. Confirmar acceso y documentar fecha de renovacion.

Tip de seguridad: para deploy en Vercel, mantén claves reales fuera del repositorio y registra solo el procedimiento de acceso.
